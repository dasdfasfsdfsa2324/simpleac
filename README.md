# SimpleAC — простой античит для Spigot/Paper 1.16.x

Плагин детектирует (эвристически, по паттернам поведения):
- **KillAura** — удары без взмаха руки, невозможный угол атаки, слишком большая дистанция, резкая смена цели
- **Fly** — зависание/полёт без легитимной причины
- **Spider** — подъём вдоль стены без лестницы/лианы
- **Baritone / боты-автоматизация** — идеально прямолинейное движение и идеально точная синхронизация взгляда с направлением движения (человек так не двигается)
- **Scaffold** — слишком частая установка блоков под ноги, "мост" без взгляда вниз
- **AutoClicker** — высокий CPS с подозрительно ровными интервалами между кликами
- **Speed** — превышение допустимой горизонтальной скорости

⚠️ Это не замена промышленным решениям (Matrix, Grim, Vulcan, NCP) — там годы тонкой настройки под протокол и NMS.
Здесь используются **только публичные Bukkit-события**, поэтому часть продвинутых читов (особенно те, что подделывают пакеты
на уровне, невидимом для Bukkit API) могут не ловиться на 100%. Для серьёзной защиты рекомендуется:
1. Использовать этот плагин как дополнение, а не единственную защиту.
2. Дать администраторам права `simpleac.alerts`, чтобы вручную проверять подозрительных игроков.
3. Аккуратно настроить `vl-to-punish` в `config.yml`, чтобы избежать ложных банов (сначала используйте только `kick`, без `ban`, понаблюдайте пару дней).
4. При возможности — поставить рядом NoCheatPlus/Grim в качестве основной защиты, а SimpleAC — как дублирующий слой на специфичные паттерны (Baritone/Scaffold).

## Сборка

Нужен установленный **Maven** и **JDK 8+**, а также доступ в интернет (для скачивания Spigot API).

```bash
cd SimpleAC
mvn clean package
```

Готовый файл появится в `target/SimpleAC.jar` — киньте его в папку `plugins` вашего сервера и перезапустите сервер.

Если у вас нет собранного Spigot API в локальном репозитории — сначала соберите BuildTools:
```bash
java -jar BuildTools.jar --rev 1.16.5
```

## Настройка

Все параметры — в `config.yml` (создаётся при первом запуске в `plugins/SimpleAC/config.yml`):
- `checks.<name>.enabled` — включить/выключить проверку
- `checks.<name>.vl-to-punish` — сколько очков нарушения нужно набрать до наказания
- `punishments.<name>` — список команд-наказаний по нарастающей (кик → бан)
- `decay` — как быстро "остывают" очки нарушений со временем (чтобы случайные лаги не приводили к бану)

## Команды

- `/anticheat reload` — перезагрузить конфиг
- `/anticheat vl <ник>` — посмотреть текущие очки нарушений игрока

## Права

- `simpleac.alerts` — получать оповещения о срабатывании чеков (по умолчанию у оп)
- `simpleac.bypass` — полностью игнорировать все проверки (выдавайте только доверенным/тестовым аккаунтам)
- `simpleac.admin` — доступ к командам `/anticheat`

## Важно про ложные срабатывания

Проверка Baritone основана на статистических паттернах (слишком идеальное движение/взгляд).
У игроков с очень стабильным пингом и аим-тренировкой возможны редкие ложные флаги — поэтому:
- начисление VL по этой проверке небольшое,
- порог `vl-to-punish` для неё выставлен выше остальных,
- первая ступень наказания — только `kick`, не `ban`.

Рекомендуется первую неделю подержать `punishments` только с kick-командами и посмотреть логи алертов,
прежде чем включать автоматические баны.
